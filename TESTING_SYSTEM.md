### 场景组 E：AI 精度极限

#### E1: 承诺提取边界测试
```
输入邮件样本（测试 precision/recall）：

真阳性（应该提取）：
- "I'll send you the proposal by Friday" → i_promised, deadline 周五
- "Can you review this by EOD?" → i_promised（收件人的承诺）
- "We will get back to you next week" → they_promised
- "老板放心，这个我周三前搞定" → i_promised, deadline 周三
- "答应 Emily 周末去动物园" → family

真阴性（不应该提取）：
- "Thanks for your help" → NOT a commitment
- "FYI, meeting notes attached" → NOT a commitment
- "Happy birthday!" → NOT a commitment
- "As discussed, here's the summary" → NOT a commitment (已完成)
- 公司群发的 newsletter → NOT a commitment

假阳性（容易误提取）：
- "I would be happy to help if needed" → 条件性，不是承诺
- "Let me know if you need anything" → 客套话
- "Looking forward to hearing from you" → 期望，不是承诺

假阴性（容易漏提取）：
- 隐含承诺："I'm on it" → i_promised（隐含"我在处理"）
- 模糊时间："sometime next month" → deadline 模糊
- 中英混合："这个 proposal 我 Friday 前 send 给你" → i_promised

验证项：
☐ Precision ≥ 85%（提取的承诺中，真的是承诺的比例）
☐ Recall ≥ 80%（实际承诺中，被提取到的比例）
☐ 家庭承诺 recall = 100%（不允许漏掉）
☐ 中英混合内容正确处理
```

```

☐ 状态从 overdue → done

```

