## Product Vision v1.0 | 2026-03-30

## 产品形态

**WhatsApp 日常交互 + Web Dashboard 看全局**

```
WhatsApp (80% 交互)          Web Dashboard (20% 全局视图)
┌──────────────────┐         ┌────────────────────────────────┐
│ 语音/文字/拍照    │         │ 承诺仪表盘 │ 关系图谱 │ 出差线 │
│ Chief 主动推送    │   ←→    │ 家庭日历   │ 周报     │ 设置   │
│ 快速确认/审批     │         │ 全局搜索   │ 历史     │ 洞察   │
└──────────────────┘         └────────────────────────────────┘
     行动层                          洞察层
  "快速做决定"                    "看全貌、做规划"
```

**分工原则：**
- WhatsApp 负责**行动**：Chief 推送 → 用户决策 → Chief 执行
- Dashboard 负责**洞察**：看全貌、做规划、查历史、调设置
- 两边实时同步，WhatsApp 里确认的事 Dashboard 立刻更新

## 一、承诺层 — 产品远景

### 1.1 承诺仪表盘 (Commitment Dashboard)

Dashboard 的**首页**不再是"Today"概览，而是**承诺仪表盘**——这是 Chief 的灵魂界面。

```
┌─────────────────────────────────────────────────────────────┐

```

**承诺卡片的核心字段：**

| 字段 | 说明 |
|------|------|
| 承诺方 | 你承诺的 / 对方承诺的 / 你对家人承诺的 |
| 对象 | 人名 + 关系标签（客户/团队/家人） |
| 内容 | 一句话描述 |
| 来源 | 从哪提取的（邮件/WhatsApp/语音/口头告知） |
| Deadline | 明确日期 or 模糊（"下周"/"尽快"） |
| 状态 | 待行动 / 进行中 / 等对方 / 已完成 / 已过期 |
| 紧急度 | Chief AI 判断（红/黄/绿） |
| 关联 | 关联的邮件/会议/任务/联系人 |

**承诺的来源（Chief 如何发现承诺）：**

```
自动提取                          手动输入
├── Gmail 邮件文本分析             ├── WhatsApp 告诉 Chief
├── WhatsApp 消息流               ├── 语音留言
├── 日历事件描述                   ├── Dashboard 手动创建
├── 会议纪要                       └── 拍照（白板/笔记）
└── 语音消息转文字
```

**承诺生命周期：**

```
发现 → 确认 → 追踪 → 提醒 → 闭环
 │      │      │      │      │
 │      │      │      │      └── Chief: "David 的方案已发送 ✅"
 │      │      │      └── Chief: "老板，David 方案明天到期"
 │      │      └── Chief 每日简报里更新进度
 │      └── Chief: "你是不是答应了 David 出方案？" → 用户确认
 └── Chief 从邮件中检测到 "I'll send you the proposal by Friday"
```

### 1.2 承诺的智能分级

Chief 不只是列清单，还要**帮你判断轻重缓急**：

| 信号 | Chief 的判断 |
|------|-------------|
| 对方是 VIP（投资人/大客户） | 紧急度 +2 |
| 已经被催过一次 | 紧急度 +2 |
| Deadline 在 24h 内 | 紧急度 +3 |
| 你答应了但对方没催 | 紧急度 +1（但仍需履约） |
| 纯 CC 的邮件里提到的 | 紧急度 -1，可能不是真承诺 |
| 对孩子的承诺 | 不分紧急度，**作为硬约束处理** |

### 1.3 承诺闭环动作

每个承诺卡片都有**一键动作**，Chief 帮你完成最后一步：

| 动作 | Chief 做什么 |
|------|-------------|
| [起草回复] | 生成邮件草稿 → WhatsApp 推给你确认 → 你说"发" → 发送 |
| [发提醒] | 生成一封礼貌的催促邮件 → 你确认 → 发送 |
| [延期] | Chief 问"延到什么时候？" → 更新 deadline → 如果对方在等，起草延期通知 |
| [已完成] | 标记完成 → 更新守约率 → 如果对方在等，通知对方 |
| [升级] | 对方长期未回 → Chief 建议换个渠道/找别人推动 |

## 三、家庭层 — 产品远景

### 3.1 家庭日历 (Family Calendar)

**数据来源：双通道输入**

```
通道 1: Google Calendar 自动读取
├── 用户在 GCal 里标记的家庭事件
├── 学校共享日历（如果学校提供 .ics）
└── 配偶共享的日历

通道 2: 口头告诉 Chief
├── WhatsApp: "暑假 6/15-8/15，别排出差"
├── WhatsApp: "每周三下午 Emily 钢琴课，我要接"
├── WhatsApp: "端午节要包粽子，记一下"
└── WhatsApp: "答应儿子周末去动物园"

### 3.2 家庭层的工作方式

家庭层**不主动管理家庭**，而是在工作决策时**自动检查家庭约束**：

| 触发场景 | Chief 的反应 |
|---------|-------------|
| 有人约你周三下午开会 | "⚠️ 周三 15:30 你要接 Emily 钢琴课，建议改时间" |
| 你准备接受一个周六的 call | "周六上午是家庭时间，确定要接吗？" |
| 安排 6 月出差超过 3 天 | "暑假 6/15 开始，你说过不安排长途出差" |
| 快到结婚纪念日 | "5/12 是结婚纪念日，要不要我帮你订餐厅？"（只提醒+建议，不代订） |
| 出差日跟孩子演出冲突 | **直接在晨间简报里预警**，不等你发现 |
| 投资人约你儿子生日那天 | "7/15 是儿子生日，建议改期。我草拟改期邮件？" |

### 3.3 家庭承诺 = 商业承诺

在承诺仪表盘里，家庭承诺和商业承诺**并列显示**，但有视觉区分：

```
⚠️ 需要你行动 (3)

🔴 投资人 Ben — 月度 update           deadline: 周三
👨‍👧 Emily — 暑假日本                    deadline: 5月底前定   ← 家庭标签
🟡 David — 合作方案                    deadline: 周五
```

**家庭承诺的特殊处理：**
- 不会被"延期"按钮轻易推迟（Chief 会多问一句："确定要推迟对 Emily 的承诺吗？"）
- 没有"取消"选项，只有"已完成"和"调整"
- 守约率统计里，家庭承诺单独计算，不跟商业承诺混在一起

| Contacts | **关系**（升级） | 加入承诺记录+品味 |
| Trips | **出差**（升级） | 活的时间线 |
| Expenses | **财务** | 改名 |
| — | **家庭**（新增） | 新维度 |
| — | **洞察**（新增） | 周报/月报/趋势 |

**导航从 8 项精简为 8 项，但每一项都围绕三维价值重新组织。**

## 五、WhatsApp 交互模式（远景）

### 5.1 Chief 的主动推送

| 推送类型 | 时机 | 内容 |
|---------|------|------|
| 晨间简报 | 每天早上（学习用户习惯后自适应时间） | 今日 3 件事 + 承诺到期提醒 + 家庭提醒 |
| 承诺到期 | deadline 前 24h / 当天 | 具体承诺 + 一键动作（起草/延期） |
| 家庭冲突 | 检测到冲突时立刻 | 冲突详情 + 改期建议 + 草稿 |
| 会前简报 | 会议前 15min | 对方档案 + 公司近况 + 上次承诺 |
| 落地简报 | 检测到到达新城市 | 城市卡 + 今日安排 + 当地信息 |
| 出差后整理 | 返回后 24h | 报销汇总 + 感谢信草稿 + 新承诺整理 |
| 周总结 | 周五下午 | 守约率 + 下周预览 + 家庭时间统计 |
| 催促建议 | 对方逾期超过设定天数 | "张总的合同审批等了 12 天，要催一下吗？" |

### 5.2 用户的快速指令

```
"我欠谁什么？"        → 列出你的 pending 承诺
"谁欠我什么？"        → 列出别人的 pending 承诺

-- 承诺表（核心）
commitments (

  recurrence,                        -- 周三钢琴课 = weekly
  source: 'google_calendar' | 'manual',

## 八、从现在到远景的演进路径

```
现在 (v1.3)                     远景
─────────────────────────────────────────────────
Today 概览          →→→         承诺仪表盘（首页）
Inbox 邮件          →→→         跨渠道统一收件箱
Tasks + Follow-ups  →→→         合并进承诺层
Contacts            →→→         关系（+承诺记录+品味）
Trips + Expenses    →→→         出差时间线 + 财务
—                   →→→         家庭日历（新增）
—                   →→→         洞察周报/月报（新增）
Settings            →→→         设置（+家庭设置+偏好学习）
```

**不需要推倒重来。** 现有的 follow_ups 表就是承诺的雏形，contacts 表已经有品味字段，trips 表已经存在。核心改造是：
1. follow_ups → 升级为 commitments（加家庭承诺、加来源追踪）
2. 新增 family_calendar 表
3. 新增 trip_timeline_events 表
4. 新增 insights_snapshots 表
5. Dashboard 首页从 Today 改为承诺仪表盘
6. 导航重组
