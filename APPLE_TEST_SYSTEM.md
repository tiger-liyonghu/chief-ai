# Apple WhatsApp 能力测试体系
## v1.0 | 2026-03-30

---

## 一、测试架构

```
测试脚本 (tests/apple/)
    │
    ├── 发送消息 → WhatsApp Service API (/api/send)
    │                    │
    │                    ▼
    │              Apple AI Engine
    │              (LLM + 32 Tools)
    │                    │
    │                    ▼
    ├── 验证回复 ← WhatsApp 回复内容
    │
    ├── 验证后台 → Supabase 数据变更
    │              (承诺/联系人/任务/出差/报销)
    │
    └── 记录指标 → 响应时间/准确率/工具调用成功率
```

**测试方式：通过WhatsApp Service的API直接发送消息，模拟用户跟Apple对话，然后检查：**
1. Apple的回复内容是否正确
2. 后台数据是否正确变更
3. 响应时间是否在3秒内（先应后回原则）

## 五、精度评估指标

### 工具选择准确率
```
测试50条真实指令，记录:
- Apple是否选对了工具
- 工具参数是否正确提取
- 目标: 工具选择准确率 > 95%
```

### 回复质量评分
```
对每条回复打分 1-5:
5 = 完美回复，准确且有用
4 = 基本正确，可以更好
3 = 部分正确，有遗漏
2 = 方向对但细节错
1 = 完全错误或无关

目标: 平均分 > 4.0
```

### 响应时间分布
```
| 操作类型 | 目标 P95 |
|---------|---------|
| 简单查询（日历/任务） | < 5秒 |
| 邮件起草 | < 10秒 |
| 先应后回（新闻查询） | 先应 < 3秒，结果 < 30秒 |
| 图片识别 | < 15秒 |
| 语音转文字 | < 10秒 |
| 晨间简报 | < 15秒 |
```

## 六、执行方式

### 自动化测试（API层）
```
通过 WhatsApp Service 的 /api/send 发送消息
等待回复（轮询 whatsapp_messages 表）
验证回复内容和后台数据变更
```

### 半自动测试（真实WhatsApp）
```
在self-chat里手动发消息
脚本监听 whatsapp_messages 表变化
自动验证后台数据
```

### 测试数据依赖
```
需要先用 seed data 填充:
  - 10+ 封邮件（含承诺）
  - 5+ 个联系人（含品味）
  - 2+ 趟出差（含航班/酒店）
  - 5+ 个承诺（各种状态）
  - 3+ 个家庭事件

用已有的 tests/seed/personas.ts 的 Jason 数据
```
