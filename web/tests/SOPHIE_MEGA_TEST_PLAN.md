# Sophie 大规模压力测试方案 V2

## 目标
通过极端复杂场景暴露 Sophie 和 App 的所有局限性，测试：
1. Sophie 在混乱中的判断力和情商
2. App 在多账号/多邮箱/多设备下的数据一致性
3. 费用/发票/出差在跨境跨币种下的准确性
4. 家庭和工作高度冲突下的冲突检测能力
5. 多种紧急情况同时发生时的优先级排序

---

## 测试矩阵

### 模块一：地狱出差（连环故障）

**场景 1.1 — 航班取消连锁反应**
```
Day 1 AM:    SQ830 SIN→PVG 取消（机械故障）
Day 1 AM+1h: 改签 SQ832 经 HKG 转机，55 分钟衔接，不同航站楼
Day 1 PM:    到上海晚 4 小时，错过华信 CTO 14:00 会议
Day 1 PM:    酒店说没收到预订（被 overbook）
Day 1 PM:    行李滞留香港
Day 1 EVE:   明天的 DBS 会议资料全在行李里的 U 盘上
```
**Sophie 应该：** 主动改约华信会议 + 找备选酒店 + 让团队从 Google Drive 拉资料

**场景 1.2 — 签证危机**
```
到 Jakarta 机场，入境官说商务签类型不对
护照被扣
合作方 CEO + 律师 + 记者在酒店等
合同必须今天签（对方董事会授权今天到期）
当地 fixer 手机打不通
```
**Sophie 应该：** 查当地大使馆紧急热线 + 建议合作方律师方案 + 起草给对方 CEO 的致歉邮件

**场景 1.3 — 跨境费用报销灾难**
```
3 周 7 国出差回来：USD, EUR, GBP, SGD, JPY, AED, IDR
公司卡在阿布扎比被冻结（银行风控），用了 $14K 个人卡
丢了 6 张出租车发票 + 2 张酒店杂费发票
一家酒店重复扣款
在东京餐厅手滑小费 $500（以为是 ¥500）
公司卡银行要求提供阿布扎比的报案证明
财务说周五前必须交报告（季度结算）
```
**测试点：** App 能否处理 7 种货币、缺失发票标记、重复扣款检测、争议标记

**场景 2.2 — 孩子受伤 + 投资人晚餐 + 生产事故**

**场景 2.3 — 春节前连环崩溃**

除夕前两天，老板在深圳飞北京

### 模块三：情绪 / 健康 / 极限

**场景 3.1 — 融资 pitch 前恐慌发作**
```
a16z 办公室，离最重要的 pitch 还有 15 分钟
老板在厕所发消息："我喘不上气，我不行了，我要取消"
合伙人没有告诉联合创始人
a16z partner 的 EA 刚发消息 "they're ready for you"
老板 38 小时没睡，今天只喝了 6 杯 espresso 没吃东西
```
**Sophie 应该：** 不说 "加油"。说具体的：深呼吸指导 + 帮买 5 分钟 + 最坏情况评估

**场景 3.2 — 在异国生病**
```
拉各斯，食物中毒，坐不住超过 5 分钟
2 小时后有 Board Call（批准 $5M bridge round，公司下个月工资靠这个）
两个董事明天飞走，3 周内无法重新安排
旅行保险卡在新加坡家里的另一个钱包
不认识当地医生，酒店前台没用
```

**场景 3.3 — 倦怠悬崖**
```
过去 2 周观察到的数据：
- 取消了 3 次个人晚餐
- 每晚睡不到 5 小时
- 凌晨 3 点发邮件
- 在 Slack 对两个下属发火（不寻常）
- 错过女儿钢琴演奏会（承诺要去的）
- 未来 9 天日历零白空间
今天说："帮我订红眼航班去纽约，投资人会议我硬扛。"
```
**Sophie 应该：** 不只是订机票。主动干预 — "你过去两周睡眠和情绪都在下降，你确定要硬扛？"

### 模块五：多账号 / 多邮箱 / 多设备复杂关系

**场景 5.1 — 多邮箱身份混乱**
```
老板有 3 个邮箱：
(a) tiger@company.com — 公司邮箱
(b) tigerliyonghu@gmail.com — 个人邮箱
(c) tiger.li@investor-updates.com — 投资人沟通专用
同一个联系人 Lisa Tan 从 (a) 和 (c) 都有邮件往来
承诺在 (a) 提取了一次，在 (c) 又提取了一次 → 重复
有些邮件从 (b) 发但 (a) 的签名 → 联系人匹配混乱
```
**测试点：** 去重、跨邮箱联系人合并、承诺不重复

**场景 5.2 — 多 WhatsApp 号码**
```
老板有 2 个 WhatsApp 号码：
(a) +65 9xxx xxxx — 新加坡号（工作）
(b) +86 1xx xxxx xxxx — 中国号（家人 + 中国客户）
两个号都连了 Chief
同一个消息在两个号都收到
中国客户只存了 (b)，投资人只存了 (a)
```
**测试点：** 消息去重、号码关联联系人、不同号码的 briefing 合并

**场景 5.3 — 账号注册 / 删除 / 重建**
```
用户 A 注册 → 连了 Gmail → 扫了 100 封邮件 → 提取了 20 个承诺
用户 A 删除账号 → 所有数据应该清除
同一个 Gmail 重新注册 → 不应该看到旧数据
用另一个 Gmail 注册 → 加了同一个 WhatsApp 号码 → 应该不影响
旧的 WhatsApp session 应该被清除
```
**测试点：** CASCADE 删除、数据隔离、session 清理

**场景 5.4 — 团队多人共用**
```
CEO 和 COO 都用 Chief
共同联系人 Lisa Tan 分别在各自的 contacts 表
CEO 标记了一个承诺完成，COO 看不到（正确行为）
但 CEO 给 COO 转发的邮件被扫描 → 提取了不属于 CEO 的承诺
```
**测试点：** 数据隔离、转发邮件的承诺归属

### 模块六：Sophie 情商 / 判断力极限

**场景 6.1 — 丧亲后的失控决策**
```
老板母亲 5 天前去世，休了 2 天就回来了
异常行为：
- 批准了之前否决的 $200K 营销预算
- 对员工因小事发 warning
- 同意了明显不利的合作条款
- 3 小时后就要签那个不利合同
下属不敢说
```
**Sophie 应该：** 识别模式异常，在签约前提醒："这个条款和你之前的立场不一致，要不要再看一眼？"

**场景 6.2 — "全完了" 的情绪崩溃**
```
跨境电商老板，凌晨 3 点连收 5 个坏消息：
(a) 亚马逊店铺因刷单嫌疑被封（200 万库存冻结）
(b) 义乌仓库发错 500 单（发了成人用品给母婴客户）
(c) PayPal 因退款率过高被限制提现（87 万美金）
(d) 美国合伙人要求退出 + 回购股份（200 万美金，30 天内）
(e) 老婆发现了他另一家不知道的公司
老板只说："完了完了全完了"
```
**Sophie 应该：** 保持冷静，按优先级拆解 5 个问题的第一步行动，先稳住情绪

## 执行方式

每个模块创建对应的种子数据脚本，然后通过以下方式测试：

1. **Dashboard Chat** — 发送场景描述，看 Sophie 回复
2. **WhatsApp** — 模拟紧急消息，看 Sophie 反应
3. **API 直接测试** — 验证数据层面的正确性（去重、隔离、级联删除）
4. **自动化脚本** — 模拟多账号注册/删除/重建流程

## 评分标准

| 维度 | 满分标准 |
|------|---------|
| 判断力 | 5 个问题同时来，能排出正确的 1-2-3 |
| 情商 | 老板崩溃时不说 "加油"，而是给具体的下一步 |
| 文化敏感 | 知道送钟的忌讳、斋月的排程、日本的道歉协议 |
| 数据准确 | 多邮箱不重复、多 WhatsApp 不混乱、删除后干净 |
| 主动性 | 不等老板问，主动发现异常（倦怠、失控决策、冲突） |
| 克制 | 没事不说话，重要的事才说 |
