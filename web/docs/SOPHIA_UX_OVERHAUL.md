# Sophia UX 大修方案

**来源：UX 研究员 + UI 设计师 + 产品经理三方会审**

---

## 一、诊断共识

三位专家独立得出相同结论：

> **Sophia 有表面没深度。问题不是功能少，是点开之后没有下一步。**

三个根因：

| 根因 | 表现 | 修复方向 |
|------|------|---------|
| **导航按数据类型而非用户意图** | Commitments/Inbox/Calendar/Contacts 是数据库表名 | 改为 Today/Calendar/Inbox，按「现在该做什么」组织 |
| **模块之间无连接** | 邮件里看不到联系人，承诺里点不到联系人详情，日历里看不到承诺 | 所有数据交叉链接 |
| **有名词无动词** | 看到邮件但没有 Send，看到承诺但点不开，看到冲突但没有 Resolve | 每个数据点都变成可操作的 |

**设计评分：C-。** 排版无层次、视觉无焦点、色彩无温度。

---

## 二、最高优先级的 5 个改变

### 1. 把首页从「承诺列表」改成「Today Briefing」

**现在：** 打开 App → 看到一堆逾期承诺 → 焦虑
**应该：** 打开 App → 看到 Sophia 的晨间简报 → 知道今天最重要的 1-2 件事

```
┌─────────────────────────────────────┐
│ Good morning, Tiger.                │
│                                     │
│ 今天最重要的事：                      │
│ 回 Lisa 的 DD checklist（逾期3天）  │
│ [📝 Draft Reply]  [✓ Mark Done]    │
│                                     │
│ 📅 14:00 投资人会议 (Lisa, Office)  │
│ ⚠️ 和 Emily 学校接送 14:45 冲突     │
│ [🔄 Reschedule] [💬 Tell Wife]     │
│                                     │
│ 💗 Emily 钢琴课今天 15:30           │
│                                     │
│ 你这周完成率 50%，有 16 件事在跑。    │
│ 建议推掉几个不紧急的。               │
└─────────────────────────────────────┘
```

每个条目可点击、可操作。不是文本，是交互卡片。

### 2. 杀死所有死角 — 点击必有下一步

| 现在 | 应该 |
|------|------|
| 承诺卡片：联系人名字是纯文本 | → 点击跳到联系人详情 |
| 邮件展开：有 AI 草稿但没有 Send | → ✅ 已修（Send Reply 按钮） |
| 日历事件：点击只展开，没有操作 | → 显示 Prep Brief + Join Meeting + Reschedule |
| 联系人：看到温度但没法操作 | → 「Send Check-in」「Schedule Catch-up」 |
| 冲突标记：只显示 ⚠️ 没有解决方案 | → 冲突解决卡片（改时间/通知家人） |

### 3. 排版层次 — 打破流水账感

**现在：** 90% 的文本是 text-xs / text-sm / text-base，三种大小几乎一样。
**应该：**

```
标题层（text-2xl font-semibold）: Good morning, Tiger
节标题（text-lg font-medium uppercase tracking-wider）: TODAY'S MEETINGS
内容层（text-sm）: 正常信息
辅助层（text-xs text-tertiary）: 时间、来源
```

一个页面至少 4 层字号。最重要的信息最大。

### 4. 色彩温度 — 从冷变暖

**现在：** 纯 indigo (#6366f1) + slate，冷且通用
**应该：**
- 主色：深navy (#1e3a5f) 或暖灰 (#2d3436)，专业但不冷
- 背景：warm white (#faf9f7) 而不是纯白
- 家庭层：coral/pink (#e17055)，有温度
- 成功：teal (#00b894)，积极
- Sophia 标识：indigo→violet 渐变，有辨识度

### 5. 三模块交叉链接

```
邮件 ←→ 联系人 ←→ 承诺 ←→ 日历

看邮件时：显示发件人的 VIP 标签 + 和这个人的承诺
看联系人时：显示互动时间线（邮件+WhatsApp+会议）
看承诺时：点联系人名跳到联系人详情
看日历时：显示会议相关的承诺 + 联系人背景
```

## 六、Sophia 视觉身份

| 方面 | 现在 | 应该 |
|------|------|------|
| 图标 | 通用 Sparkles ✨ | 定制 "S" monogram，indigo→violet 渐变 |
| 出现位置 | 只在 chat 面板 | 每个页面都有 Sophia 的一句话（insight bar） |
| 聊天入口 | 桌面右上角小按钮，手机看不到 | 所有页面底部悬浮按钮（手机也有） |
| 通知 | 无 | Toast 通知：「Lisa 的邮件等了 3 天了，要我起草回复吗？」 |
| 语气可视化 | 无 | 情绪检测后微调 UI 色温（用户疲劳时背景更暖） |

## 七、执行路线图

### Phase 1（2 周）— 杀死死角 + 基础体验

| 任务 | 影响 |
|------|------|
| Today Briefing 首页（替代承诺列表） | 第一印象跳变 |
| 邮件线程视图 + 附件 + 搜索 | 邮箱可用 |
| 日历事件详情页（Join/Prep/Reschedule） | 日历可用 |
| 联系人互动时间线 | 人脉可用 |
| 所有模块交叉链接 | 消灭孤岛 |
| 会议链接提取（Teams/Zoom/Meet） | 关键操作 |

### Phase 2（2 周）— 设计升级 + 智能

| 任务 | 影响 |
|------|------|
| 排版层次重建（4 层字号） | 打破流水账 |
| 色彩温度调整 | 专业+温暖 |

| 邮件智能分类（需回复/FYI） | 减少认知负荷 |

### Phase 3（2 周）— 差异化

| 任务 | 影响 |
|------|------|

## 九、核心设计原则

1. **Advisor First** — 每个屏幕是一次对话，不是一张表格
2. **Context Over Lists** — 不显示孤立数据，显示关联和原因
3. **Time is Master** — 一切围绕时间轴组织
4. **Family is Sacred** — 家庭不是单独 tab，是渗透在每个视图里的温暖色彩
5. **Progressive Disclosure** — 先显示 1 件事，点开才显示全部
6. **Warm Professional** — 像皮质笔记本，不像 Excel
