## Product Blueprint v1.0 | 2026-03-29

---

## 一句话定位

> **Apple 是你口袋里的资深秘书** — 通过 WhatsApp/Telegram 跟你对话，管你的日程、邮件、客户关系、出差全流程、发票报销，像一个跟了你三年的真人幕僚长。

## 为什么是现在

1. **AI 能力拐点** — LLM 现在能可靠地跨语言理解邮件、提取任务、起草回复、识别发票。18 个月前做不到。
2. **渠道碎片化到了临界点** — 知识工作者日均 4-6 个通讯工具，没有人拥有"一个大脑管所有渠道"的位置。
3. **竞品窗口 6-9 个月** — Kinso、OpenClaw、HeyRobyn 都在跑，但没人做到"WhatsApp 原生 + 差旅智能 + 客户情报"。
4. **全球顶级 EA 的工作方式可以被产品化** — 她们手动在 8-12 个工具之间当集成层，Apple 就是那个自动化的集成层。

## Apple 的时间应对策略

Apple 不是所有事都秒回。像真人秘书一样，有些事需要时间去查、去研究。关键是**不让老板等着没回音**。

| 响应类型 | 时间 | 适用场景 | Apple 行为 |
|---------|------|---------|-----------|
| **秒回** | < 3 秒 | 查已有数据（日程、任务、联系人） | 直接回答 |
| **快回** | < 30 秒 | 需要调 API 或简单推理（起草邮件） | 直接回答或生成草稿 |
| **先应后回** | 1-10 分钟 | 需要研究（公司新闻、行业情报、复杂分析） | 先说"收到老板，我查一下" → 研究完推送结果 |
| **定时推** | 定时 | 晨间简报、出差前整理、逾期提醒 | 按时主动推送，不需要老板问 |
| **异步确认** | 等老板回 | 邮件草稿需确认、报销单需审核 | 推送草稿 → 等老板说"发" → 执行 |

**核心原则：老板发了消息，3 秒内必有响应。要么是答案，要么是"收到，我去查"。绝不能沉默。**

**先应后回示例：**
```
老板：帮我查一下 Maybank 最近有什么新闻

🍎 好的老板，我去查一下 Maybank 的近况。

（3 分钟后）

🍎 查好了，Maybank 近期 3 件事：
1. Q3 净利润下滑 12%，CEO 上周内部讲话提到"控制成本"
2. 投行部新 Head 上任，从 Goldman 过来的
3. 和 Grab 合作推数字银行，下月上线
明天见李明可以聊第 3 点，跟你的业务相关。
```

### 圈 3：关系 — 客户情报中心

Apple 帮你经营 200+ 商务关系，像顶级 EA 维护的个人 CRM。

**名片管理：**
- 拍名片自动入库 → 关联公司档案
- 去重检测："张总从 CIMB 跳到 Maybank 了"
- 标注认识场景："3/31 KL 会议上张总介绍的"

**客户档案（公司级）：**
- 公司基本面：行业、规模、总部、主营
- 组织架构：关键部门和汇报线
- 你在这家公司认识谁、跟谁有过交互
- 合作历史：邮件/会议/跟进汇总

**高管档案（个人级）：**
- 职位、背景、履历
- 你跟他的所有交互记录（邮件、会议、WhatsApp）
- 个人偏好："不喝酒"、"喜欢高尔夫"、"配偶叫 Sarah"
- 上次见面聊了什么、你答应了什么

**"你在这个城市认识谁"：**
- 联系人按城市打标签
- 出差前自动查："老板，你在 KL 认识 5 个人，张总和李明上次见面超过 3 个月了，要不要约？"

**交互示例：**

## Apple 的进化引擎

Apple 不是静态工具。她通过三层同步优化，越用越强。

### 三层优化架构

```
┌─────────────────────────────────────────────────────────┐
│  第三层：平台进化（我们后台）                 慢，季度级  │
│  模型升级、新能力、新渠道、新数据源                       │
├─────────────────────────────────────────────────────────┤
│  第二层：群体智慧（跨用户匿名学习）           中，周级    │
│  哪些推荐被采纳、哪些简报格式打开率高                     │
├─────────────────────────────────────────────────────────┤
│  第一层：个人磨合（Apple 跟这个老板）         快，每次对话 │
│  偏好、习惯、关系、品味、节奏                             │
└─────────────────────────────────────────────────────────┘
```

**权重：个人磨合 70% > 群体智慧 20% > 平台进化 10%**

起步阶段几乎 100% 靠第一层。个人磨合是唯一能让第一批用户留下来的东西。

### 第一层：个人磨合（最关键）

不靠老板填表，靠**日常互动中自动学习**。

**三种学习方式：**

**① 显性反馈 — 老板直接说：**
```
老板：以后简报不要超过 5 行
🍎 收到，以后简报控制在 5 行以内。

老板：张总不吃海鲜，记一下
🍎 好的，已标注张总忌海鲜。
```

**② 隐性反馈 — 从行为推断：**

| Apple 观察到的 | Apple 学到的 |
|---------------|------------|
| 老板每次都改草稿的开头称呼 | 下次直接用老板习惯的称呼方式 |
| 推荐了 3 家餐厅，老板总选日料 | 偏好日料，优先推 |
| 晨间简报 8:00 推，老板 8:45 才看 | 改成 8:30 推 |
| 老板从不用"导出 PDF"功能 | 降低这个建议的频率 |
| 老板总在周五下午问"下周有什么" | 周五 3 点主动推下周预览 |
| 邮件草稿老板总把语气改得更直接 | 学习老板的写作风格，下次直接写直接 |
| 老板拒绝了红眼航班的推荐 | 标记偏好：不坐红眼 |

**③ 主动确认 — 不确定时问一次：**
```
🍎 老板，我注意到你最近 3 次都选了靠走道的位子。
以后帮你默认选走道可以吗？

老板：可以
🍎 好的，以后机票默认走道。
```

**核心原则：观察 3 次，确认 1 次，之后永远记住。不反复问。**

**磨合维度清单：**

| 维度 | 学什么 | 怎么学 |
|------|--------|--------|
| 沟通风格 | 简洁 vs 详细、语言、称呼习惯 | 老板改草稿的模式 |
| 时间节奏 | 几点看简报、几点处理邮件、周几最忙 | 消息打开时间分析 |
| 决策偏好 | 给选项 vs 直接建议、要理由 vs 要结论 | 老板的回复模式 |
| 优先级判断 | 谁是 VIP、什么事情可以等 | 老板的回复速度和处理顺序 |
| 品味偏好 | 吃什么、住哪、怎么飞 | 选择/拒绝记录 |
| 写作风格 | 用词、语气、格式、签名 | 老板对草稿的修改 |
| 精力曲线 | 什么时候高效、什么时候需要休息 | 活跃时间和消息频率 |
| 关系权重 | 谁重要、谁可以慢回 | 老板对不同人的响应速度 |

**磨合时间线：**
```
第 1 周：Apple 还比较生硬，像新来的秘书
         ↓  学会语言偏好、基本作息、核心联系人
第 2-4 周：Apple 开始主动，不用老板问就推简报
         ↓  学会写作风格、品味偏好、优先级判断
第 2-3 月：Apple 像跟了半年的秘书
         ↓  知道什么事老板要亲自处理、什么事可以代劳
第 6 月+：Apple 像跟了三年的秘书
         ↓  能预判老板的需求、挡掉不该打扰的事
```

---

### 第二层：群体智慧

跨用户的匿名聚合学习，**绝不涉及个人内容**。

| 我们学什么 | 怎么用 |
|-----------|--------|
| KL 排名前 10 的商务宴请餐厅（基于所有用户的选择） | 新用户去 KL 时有更好的默认推荐 |
| 晨间简报哪种格式打开率最高 | 优化默认简报模板 |
| 哪些行业情报推送被点开、哪些被忽略 | 调整情报筛选算法 |
| 出差报销最常漏掉的票据类型 | 优化缺票检查逻辑 |
| 某个城市的 Grab vs 出租车价格比 | 更准确的交通建议 |
| 不同文化背景用户的邮件风格差异 | 更好的多语言草稿能力 |
| 哪些"先应后回"的任务实际处理时间多长 | 更准确的"稍后回复"时间估计 |

**隐私红线：只聚合统计数据（"80% 的用户选了走道"），绝不跨用户共享个人内容。**

```
个人磨合（Apple 越来越懂这个老板）
    ↓ 老板用得越多

    ↓ 默认体验越好
新用户上手越快
    ↓ 更多用户
更多磨合数据 → Apple 越来越强
    ↓
网络效应：用 Apple 的高管越多

→ 每个用户受益
```

## 数据护城河

越用越深的壁垒，竞品无法复制：

```
用户使用 1 个月 → 联系人图谱建立 → 交互历史积累
                                        ↓
用户使用 3 个月 → 偏好引擎成熟 → Apple 知道你的习惯
                                        ↓
用户使用 6 个月 → 客户情报深化 → 关系网络不可迁移
                                        ↓
用户使用 1 年  → 完整的商务记忆 → 换工具 = 失忆
```

四道壁垒：
1. **统一联系人图谱** — 同一个人跨 Gmail、WhatsApp、Calendar 的所有交互串起来
2. **偏好引擎** — 知道你的精力曲线、饮食偏好、座位偏好、作息时间
3. **关系记忆** — 谁是 VIP、你欠谁什么、上次跟谁聊了什么
4. **品味档案** — 老板和每个客户的吃喝玩乐偏好，越积越厚
